# Neuraclinic Auth Microservice

Go gRPC authentication service for Neuraclinic.

## Requirements

- Go 1.25+
- Docker and Docker Compose
- OpenSSL for local TLS and JWT keys
- Buf if protobuf code needs regeneration

## Local Setup

Run from `neuraclinic-auth`:

```bash
make create-envs
make jwt-keys
make tls-generate-dev
make create-network
make compose-build
```

The service listens inside Docker on `:8000` and is exposed on host port `8002`.

`neuraclinic-users` must be running on `neuraclinic-network` because auth calls `UserService` over gRPC.

## Security Notes

- `INTERNAL_SERVICE_TOKEN` must match the value configured in `neuraclinic-users`.
- Access and refresh JWTs use separate RSA key pairs.
- Password reset OTPs and reset tokens are stored hashed.
- In development, the reset OTP is logged so flows can be tested before Notifications is ready.

## ID Convention

- All new IDs generated by this service must use `uuidv7` to stay aligned with `neuraclinic-users` and the rest of the platform.
- PostgreSQL stores these values in `uuid` columns. No schema change is required to support `uuidv7`.
- IDs that come from fixed catalog data, such as seeded permissions, can remain static UUID literals.
